# Task: AI Health Report & Diet Recommendation System

Implement a cross-service feature for "AI 식단 추천" (AI Diet Recommendation) that analyzes user habits and fridge inventory to generate a health report.

## 1. System Architecture
- **Frontend (Next.js):** Triggers the request, handles async loading state, and displays the final report.
- **Backend 1 (Spring Boot):** Orchestrates data. Fetches meal history and fridge data from DB, calls Backend 2, saves the AI response to the DB, and returns a DTO.
- **Backend 2 (FastAPI):** Acts as the AI agent. Uses `gemini-2.5-flash` to process data and return a structured JSON response.

## 2. Feature Requirements

### A. Data Gathering (Spring Boot)
- Retrieve the current user's fridge ingredients (`UserIngredient`).
- Retrieve the user's recent meal logs (`Meal`).
- Forward this data as context to the FastAPI endpoint.

### B. AI Logic (FastAPI)
- **Model:** Use `gemini-2.5-flash` via `GEMINI_API_KEY`.
- **Constraint:** Recommended recipes MUST strictly use ingredients found in the provided fridge data.
- **Output Structure (JSON):**
    - `summary`: General statement on current eating habits.
    - `advice`: List of what to eat more of and what to eat less of.
    - `meals`: 3 recipes (Breakfast, Lunch, Dinner).
    - `videos`: 3 YouTube links/search terms for alternatives for each meal.

### C. Persistence & API (Spring Boot)
- **Entity:** Create `HealthReport` entity to store the timestamped report.
- **Controller:** Endpoint `/api/v1/health-report/generate` (POST).
- **Service:** Logic to call FastAPI, save the result, and return `HealthReportResponse` DTO.

### D. UI/UX (Next.js)
- Create a `HealthReport` component.
- Implement an async button `AI 식단 추천`.
- Show a skeleton loader while the request is in progress.
- Render the report segments: Summary, Nutritional Advice, Recipes, and Video links.
