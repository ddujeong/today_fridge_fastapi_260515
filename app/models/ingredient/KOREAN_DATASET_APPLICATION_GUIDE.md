# 국내 공개 이미지·학습데이터 이용·승인 안내

법적 효력은 **각 포털의 최신 이용약관·데이터별 제공 조건**이 우선입니다. 아래는 일반적인 절차 정리입니다.

---

## 0. 공통으로 할 일

| 항목 | 내용 |
|------|------|
| **이용 주체** | 개인 / 학교 / 회사(법인) 중 무엇으로 신청할지 정하기. 법인이면 사업자·담당자 정보가 필요한 경우가 많음. |
| **이용 목적** | 「AI 모델 학습」「졸업 프로젝트」「사내 서비스 개발」 등 구체적으로 적어 두기. 신청서·활용목적란에 그대로 쓰는 경우가 많음. |
| **보관** | 다운로드한 원본·가공본은 **외부 유출 금지**인 경우가 많음. 팀 내부 저장소·접근 권한 정리. |
| **표기** | 데이터셋마다 **출처 표기 문구**(예: NIA AI-Hub, 공공데이터포털)가 정해져 있으면 문서·모델 카드에 남기기. |

## 1. NIA AI-Hub (`aihub.or.kr`)

### 1-1. 회원

- **회원가입** 후 로그인(일부 소셜 로그인 지원).
- 상세 데이터마다 **「내국인만 신청 가능」** 등 조건이 붙는 경우가 있음 → 해당 데이터 **상세 페이지**의 안내를 반드시 확인.

### 1-2. 데이터 유형별로 갈림 (가장 중요)

**A. 일반 제공(다운로드 / 샘플 + 전체 신청)**  

- 데이터 **상세 페이지**에서 **「이용신청」「다운로드」** 등 버튼이 있음.
- 흐름 예: 로그인 → 데이터 선택 → **이용 목적·약관 동의·서약** → 승인(또는 자동 처리) → **다운로드 링크·기간** 안내.
- **이용정책**: [이용약관·가이드](https://aihub.or.kr/intrcn/guid/usagepolicy.do) — 학습용 목적, **NIA·사업 결과 표기**, 데이터 **재가공·제3자 제공·무단 배포 제한**, **국외 반출** 시 별도 조건 등이 자주 명시됨.

**B. 온라인 안심존(원격 분석 환경)**  

- 데이터가 **「안심존」**으로 분류되면, PC에 바로 내려받지 않고 **가상 서버(폐쇄망)** 안에서만 분석하는 방식인 경우가 많음.
- **신청·심의가 무겁다**: 소속 증빙, **연구계획서**, 경우에 따라 **IRB(기관생명윤리위) 또는 면제 서류**, 보안서약 등.
- 심의 기간은 **수일~2주+** 수준으로 안내되는 경우가 있음.
- 데이터셋 페이지의 **「안내」「문의」** 또는 AI-Hub **고객지원·FAQ**를 따름.
- 안심존 소개: [안심존이란?](https://aihub.or.kr/intrcn/safetyzoneintrcn.do)

### 1-3. 실무 순서 (일반 다운로드형 가정)

1. [데이터 찾기](https://aihub.or.kr/aihubdata/data/list.do)에서 키워드 검색(음식, 식재료, 작물 등).  
2. 프로젝트용으로 쓸 **데이터 상세** 들어가기.  
3. **구분(일반/안심존)**, **파일 크기**, **라벨 형식** 확인.  
4. **이용신청** → 약관·서약 → 승인 대기(있을 경우).  
5. **다운로드** 또는 **샘플 먼저 받아** 폴더 구조·라벨 확인.  
6. 팀 규칙에 맞게 `data_sources/…` 에 압축 해제 후, `normalized_name` ↔ 클래스 매핑.

### 1-4. 문의

- AI-Hub **문의하기·FAQ**([고객지원](https://aihub.or.kr/partcptnmlrd/inqry/list.do)) — 데이터별 문의처가 상세 페이지에 있을 수 있음.

## 2. 공공데이터포털 (`data.go.kr`)

### 2-1. 회원

- **회원가입** 후 로그인(필요 시).

### 2-2. 파일 데이터 vs 오픈API

| 유형 | 대략적인 이용 방법 |
|------|-------------------|
| **파일 데이터** | 상세 페이지에서 **바로 다운로드**되거나, **간단한 동의 후** 다운로드되는 경우가 많음. 데이터마다 **「다운로드」** 버튼·안내 확인. |
| **오픈 API** | **활용 신청** → 활용 목적 입력 → (자동)승인 후 **인증키** 발급 → API 호출로 수집. 대량 이미지는 파일 데이터가 나을 수 있음. |

### 2-3. 꼭 확인할 것

- 각 데이터의 **「공공데이터 이용약관」**, **「제공 기관」**, **「갱신 주기」**, **「이용허락범위」**(상업 이용·2차 저작물 등).
- 문의: **공공데이터활용지원센터** 등 포털 하단 안내(대표 번호는 시점마다 변경되므로 포털에서 확인).

## 3. 농림축산식품 공공데이터 (`data.mafra.go.kr`)

- **회원가입** 후 검색.
- 품목·병해·축산·스마트팜 등 **데이터마다 제공 형식**(파일/API)·**이용 조건**이 다름.
- 상세 페이지의 **다운로드·신청·담당 기관 문의** 안내를 따름.

## 4. 프로젝트와 맞추기 (승인 후)

1. 압축 해제 경로는 `app/models/ingredient/data/korean_public_image_datasets.json` 의 `suggested_local_dir` 를 참고.  
2. 데이터셋 클래스명 → `ingredient_master.normalized_name` → `ing_XXXXX` 매핑 표 작성.  
3. `ingest_verified_path_manifest.py` 또는 `team_uploads/train/ing_XXXXX/` 에 반영 후 `assemble` 재실행.

---

## 5. 자주 하는 오해

- **「공개」≠ 「무제한 배포」** — 대부분 **학습·연구·서비스** 범위와 **재배포 금지**가 함께 적힘.  
- **안심존 ≠ 일반 다운로드** — 절차·기간이 완전히 다름. 일정 잡을 때 안심존 여부를 먼저 볼 것.  
- **팀 과제**면 **한 명이 신청·동의**하고 파일을 팀 공유 저장소에만 두는지, **기관 단위 신청**이 필요한지 데이터마다 다를 수 있음.
